Register Your Location and Files

Direct your browser to the HTTP location of your Admin Tool, log in, and perform the following steps.

  1. Select Server

    In the Server section we create the Session Group.

    Create a new Session Group, play_group

    1. Click on the Session Group link.
    2. Fill in the name, "play_group".
    3. Click the submit button.

  2. Register the directory location with the ICE server

    Remember, the location for our "play_unit" is the directory "plays" in your chosen directory on the local file system.
    1. Select the Server Link
    2. Click the Locations link
    3. Fill in the server name for the location, "play_location".
    4. Select the ICE Location radio button
    5. Ensure that the "public" check box is not selected.
    6. Fill in the full pathname of your directory.
    7. For now we will leave the extensions entry field empty.
    8. Submit the form.

  3. Select Business Unit Environment

    In the Business Unit Envrionment section we create Business Units and assign locations and pages to the Business Units.
    • Click the menu icon in the top frame to return to the Administration Tool main menu.

    This uses an expanding tree style navigation interface. You navigate to the section you need by clicking the signs to expand the tree. To create a new instance of the selected object you click the underlined section name. For example, to create a new business unit you click the Business Units link. Once you have sucessfully created the new unit you will see it listed under the Business Unit heading.

  4. Create a new Business Unit, play_unit

    1. Click on the Business Units link.
    2. Fill in the name, "play_unit".
    3. Click the submit button.
    Notice that if you expand the Business Units the new play_unit appears.

  5. Register the Location for play_unit

    Remember, the location for our "play_unit" is the directory "plays" in your chosen directory on the local file system.
    1. Expand the Business Units sub-tree, if you haven't already.
    2. Expand the play_unit sub-tree.
    3. click locations.

  6. Register the Files for play_unit

    1. Business Unit select ''
    2. plays select ''
    3. pages
    4. Select the "take from the file system" storage type.
    5. Fill in the file name "select_plays.html".
    6. Fill in the ICE Document name "select_plays.html" (we use the same name for simplicity).
    7. Select the "iceplays" location.
    8. Submit.

  7. Create a database connection

    1. Security.
    2. Database Connections.
    3. Name: play_database.
    4. Database name: icetutor.
    5. Database user: icedefdb.
    6. Comment: "Database dedicated to the works of the Bard of Complete upon Works".
    7. Submit.

  8. Create a Role for AutoDeclare

    We need to create a role that can be assigned to the default user for when a user declares an account for themselves.
    1. Click the Security link.
    2. Click Profiles.
    3. Name: enter "play_tutor_profile".
    4. Timeout: 300
    5. Leave all other fields for now (you can always modify them later by clicking the play_tutor_profile profile.
    6. Submit the form.

  9. Associate the new play role with the play profile

    1. Security.
    2. Profiles.
    3. roles.
    4. Select "play_role" from the combo box.
    5. Submit.

  10. Associate the new play database connection with the play profile

    1. Security.
    2. Profiles.
    3. database connections.
    4. select "play_database" from the combo box.
    5. Submit.
