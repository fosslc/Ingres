1.0 Overview

This document explains how and when to make changes to the Ingres Timezone files using time zone changes in North America (2007) and Australia (2006) as examples.

Adjusting an Ingres installation for the revised Daylight Saving Time (DST) of North America (effective 11th March, 2007) and Western Australia (effective now) is required if the following Timezones are used. This document is relevant to all operating systems for which a supported Ingres release exists.

In our example the following Timezones have changed.

North America

alaska

atlantic

central

eastern

mountain

nwfdlnd

pacific

yukon

Western Australia

auswest

2.0 Updating Your Installation

2.1 Using Timezone Tool iizic

Ingres has made a new World Timezone file available that can be used to update all supported Ingres releases on all platforms. The updates can be applied to your installation by updating the Timezone files from the Timezone Data File iiworld.tz using the Ingres Timezone tool iizic. To verify the update was successful, use the Ingres Timezone tool iizck. For Hybrid Releases of Ingres either a 64bit or 32bit version of iizic is required in addition to the one included with the distribution.

2.2 Using Timezone Tool iizck

The iizck Timezone tool can be used to check the current settings of the Timezone files for the Timezones that have changed, before and after any update is performed. For Hybrid Releases of Ingres either a 64bit or 32bit version of iizck is required in addition to the one included with the distribution.

2.4 Updating the Installation Timezone Files

The iizic Timezone tool is used to convert the Timezone data file into the installation's Timezone files. The tool can be used to replace all of the Timezone files or, by providing an alternative directory, only specific files.  Note: additional actions are required if only a select number of files are to be changed.

For platforms that are listed in Table 1 above, the 64 bit Timezone files are located in the lp64 directory under the standard location of the Timezone files. To replace these, the 64bit version of the Timezone tools must be used. These need to be run with the full path name.

E.g. $II_SYSTEM/ingres/utility/lp64/iizic

For platforms that are listed in Table 2 above, the 32 bit Timezone files are located in the lp32 directory under the standard location of the Timezone files. To replace these, the 32bit version of the Timezone tools must be used. These need to be run with the full path name.

E.g. $II_SYSTEM/ingres/utility/lp32/iizic

3.0 Replacing All Timezone Files

To replace all Timezone files

  1. Open a command window or shell which has the Ingres environment configured for the Ingres installation to be patched and change the current directory to the Ingres System directory: 
    Windows     - cd %II_SYSTEM%\ingres
    
Unix/Linux  - cd $II_SYSTEM/ingres
    
VMS         - set default II_SYSTEM:[ingres]

  2. Create a temporary working directory.

    Create a temporary directory into which the Timezone data file (iiworld.tz) will be copied. This directory and contents can be deleted once all steps are completed. 

    Windows     -   mkdir temptz
    
Unix/Linux  -   mkdir temptz
    
VMS         -   create/directory [.TEMPTZ]

    Unix/Linux platforms only. For the platforms which also have 64bit or 32bit Timezone files: 

    Unix        -   mkdir temptz/lp64

    or 

    Unix/Linux  -   mkdir temptz/lp32

  3. New Timezone data file

    Place the new Timezone data file in the above directory, this will be used by all versions of the iizic Timezone tool.

    Unix platforms only. For the platforms which also have 64bit or 32bit Timezone files.

    Download the new Timezone tools (iizic and iizck) from the Ingres website to the required directory if they are not already present:

    Download Solaris (SPARC64) tools su9.us5

    Download HP-UX (PA-RISC) tools hp2.us5

    Download IBM-AIX tools r64.us5

    to $II_SYSTEM/ingres/utility/lp64

    or

    Download Linux(x86_64) tools a64.lnx

    Download Linux(Itanium) tools i64.lnx

    Download HP-UX(Itanium)tools i64.hpu

    to $II_SYSTEM/ingres/utility/lp32

    It will be necessary to set the protection on these tools if they have been downloaded. The following commands can be used to achieve this. 

    chmod 755 $II_SYSTEM/ingres/utility/lp64/iizck

chmod 755 $II_SYSTEM/ingres/utility/lp64/iizic

    or 

    chmod 755 $II_SYSTEM/ingres/utility/lp32/iizck

chmod 755 $II_SYSTEM/ingres/utility/lp32/iizic

  4. Stop Ingres.

    Shutdown the Ingres installation using ingstop, or if using the Ingres Visual Manager, then Ingres can be stopped in the normal way by clicking on the Red traffic light icon.

  5. Save ALL Timzeone files.

    These are found at: 

    Windows     - %II_SYSTEM%\ingres\files\zoneinfo\*\*

Unix/Linux  - $II_SYSTEM/ingres/files/zoneinfo/*/*

VMS         - II_SYSTEM:[ingres.files.zoneinfo...]*.;

    We suggest that these files be backed up by creating an archive file: 

    Windows     - Use Windows tools to create a zip file of the 
    
              %II_SYSTEM%\ingres\files\zoneinfo

Unix/Linux  - tar cvf zoneinfo.tar $II_SYSTEM/ingres/files/zoneinfo

VMS         - backup II_SYSTEM:[ingres.files.zoneinfo...]*.; zoneinfo.bck/sav

  6. Update Timezone files.

    To update ALL Timezone files, use the iizic Timezone tool with the Timezone data file. 

    Windows     - iizic iiworld.tz

Unix/Linux  - iizic iiworld.tz

VMS         - iizic iiworld.tz

    Unix/Linux platforms only. For the platforms which also have 64bit or 32bit Timezone files:

    To update ALL of the 64bit or 32bit Timezone files, use the iizic Timezone tool with the Timezone data file. 

    $II_SYSTEM/ingres/utility/lp64/iizic iiworld.tz

    or 

    $II_SYSTEM/ingres/utility/lp32/iizic iiworld.tz

  7. Check that the changes are good.

    Using the iizck Timezone tool, confirm that the changes have been applied correctly.

    The example here is for the AUSWEST Timezone changes. The same would apply for each of the changed NORTH-AMERICA (NA) Timezone files. 

    Windows     - iizck -f %II_SYSTEM%\ingres\files\zoneinfo\astrl\auswest

Unix/Linux  - iizck -f $II_SYSTEM/ingres/files/zoneinfo/astrl/auswest

Unix 64bit  - $II_SYSTEM/ingres/utility/lp64/iizck -f $II_SYSTEM/ingres/files/zoneinfo/lp64/astrl/auswest

Unix/Linux 32bit - $II_SYSTEM/ingres/utility/lp32/iizck -f $II_SYSTEM/ingres/files/zoneinfo/lp32/astrl/auswest

VMS         - iizck -f II_SYSTEM:[ingres.files.zoneinfo.astrl]auswest.;

4.0 Selective Update of Timezone Files

Should there be a need to replace only the essential Timezone files, then using the same steps as above (Steps 1-5) and the following additional steps will allow individual Timezone files to be replaced.

  1. Initial setup for selectively updating Timezone files

    Follow steps 1 to 5 above.

  2. Create the new Timezone files

    To write Timezone files to a separate directory structure so as not to overwrite the existing ones use the following iizic command. 

    iizic {\b–}d directoryname iiworld.tz

    where directoryname is the full path to the directory that was created in Step 2. 

    Windows     - iizic –d %II_SYSTEM%\ingres\temptz  iiworld.tz

Unix/Linux  - iizic –d $II_SYSTEM/ingres/temptz   iiworld.tz

VMS         - iizic –d II_SYSTEM:[ingres.temptz]  iiworld.tz

    Unix/Linux platforms only. For the platforms which also have 64bit or 32bit Timezone files. 

    $II_SYSTEM/ingres/utility/lp64/iizic –d directoryname64 iiworld.tz

    or 

    $II_SYSTEM/ingres/utility/lp32/iizic –d directoryname32 iiworld.tz

    where directorynameNN is the full path to an alternative 64bit/32bit directory that was created in Step 2.

  3. Check that the changes are correct

    Using the iizck Timezone tool, look at the files which were created.

    The example here is for the AUSWEST Timezone changes. The same would apply for each of the changed NORTH-AMERICA (NA) Timezone files. 

    Windows     - iizck -f %II_SYSTEM%\ingres\temptz\astrl\auswest

Unix/Linux  - iizck -f $II_SYSTEM/ingres/temptz/astrl/auswest

Unix 64bit  - $II_SYSTEM/ingres/utility/lp64/iizck -f $II_SYSTEM/ingres/files/zoneinfo/lp64/astrl/auswest

Unix/Linux 32bit - $II_SYSTEM/ingres/utility/lp32/iizck -f $II_SYSTEM/ingres/files/zoneinfo/lp32/astrl/auswest

VMS         - iizck -f II_SYSTEM:[ingres.temptz.astrl]auswest.;

  4. Select the new Timezone files to use

    Change to the directoryname directory and copy the required files to the system location. Note: for the Hybrid releases of Ingres where either the 64bit or 32bit versions of the files also exist, these must also be copied, replacing the directoryname with directoryname64 and setting the appropriate target location.

    To copy the North American files: 

    Windows     - cp %II_SYSTEM%\ingres\temptz\na\*  %II_SYSTEM%\ingres\files\zoneinfo\na\

Unix/Linux  - cp $II_SYSTEM/ingres/temptz/na/*  $II_SYSTEM/ingres/files/zoneinfo/na/

Unix 64 bit – cp $II_SYSTEM/ingres/temptz/lp64/na/*  $II_SYSTEM/ingres/files/zoneinfo/lp64/na

Linux/Unix 32bit - cp $II_SYSTEM/ingres/temptz/lp32/na/*  $II_SYSTEM/ingres/files/zoneinfo/lp32/na/

VMS         - copy II_SYSTEML[ingres.temptz.na]*.* II_SYSTEM:[ingres.files.zoneinfo.na]

    To copy the changed Australian West file: 

    Windows     - cp %II_SYSTEM%\ingres\temptz\astrl\auswest %II_SYSTEM%\ingres\files\zoneinfo\astrl\auswest

Unix/Linux  - cp $II_SYSTEM/ingres/temptz/astrl/auswest $II_SYSTEM/ingres/files/zoneinfo/astrl/auswest

Unix 64 bit – cp $II_SYSTEM/ingres/temptz/lp64/astrl/auswest $II_SYSTEM/ingres/files/zoneinfo/lp64/astrl/auswest

Linux/Unix 32bit  - cp $II_SYSTEM/ingres/temptz/lp32/astrl/auswest $II_SYSTEM/ingres/files/zoneinfo/lp32/astrl/auswest

VMS         - copy II_SYSTEML[ingres.temptz.astrl]auswest II_SYSTEM:[ingres.files.zoneinfo.astrl]auswest.;

  5. Restart Ingres

    Use ingstart to restart Ingres, or if using Ingres Visual Manager, then Ingres can be restarted in the normal way by clicking on the green traffic light icon.

  6. Clean-up

    Delete the temporary directory and contents.

    You may wish to keep this directory and the Timezone data file for reference.
